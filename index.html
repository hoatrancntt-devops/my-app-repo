Triển Khai Nginx Ingress & AWS NLB

Mục tiêu:

Thay vì dùng ALB (Layer 7) cho từng ứng dụng, ta dùng mô hình tối ưu hơn: AWS NLB (Layer 4) đứng trước để chịu tải và giải mã SSL, sau đó chuyển traffic vào Nginx Ingress bên trong cụm EKS.

7.1. Cài đặt Nginx Ingress Controller (Kèm NLB)

Sử dụng Helm để cài đặt. Lưu ý các tham số quan trọng để AWS biết tạo NLB và gắn chứng chỉ ACM.

:: 1. Thêm Repo Helm
helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx
helm repo update

:: 2. Cài đặt Nginx Ingress (Thay ARN của bạn vào dòng dưới)
helm install nginx-ingress ingress-nginx/ingress-nginx ^
  --namespace ingress-nginx ^
  --create-namespace ^
  --set controller.service.type=LoadBalancer ^
  --set controller.service.annotations."service.beta.kubernetes.io/aws-load-balancer-type"="nlb" ^
  --set controller.service.annotations."service.beta.kubernetes.io/aws-load-balancer-scheme"="internet-facing" ^
  --set controller.service.annotations."service.beta.kubernetes.io/aws-load-balancer-ssl-cert"="arn:aws:acm:ap-southeast-1:342857646954:certificate/e20cdc83-38cb-4701-b7de-bbbdcd6e5652" ^
  --set controller.service.annotations."service.beta.kubernetes.io/aws-load-balancer-ssl-ports"="443" ^
  --set controller.service.targetPorts.https=http

Giải thích tham số:

  • aws-load-balancer-type="nlb": Yêu cầu AWS tạo Network Load Balancer.
  • ssl-cert: Gắn chứng chỉ SSL vào NLB để giải mã HTTPS tại cửa ngõ (SSL Offloading).
  • targetPorts.https=http: Chuyển traffic đã giải mã (HTTP port 80) vào Nginx để giảm tải xử lý.

7.2. Cấu hình Security Group (BẮT BUỘC)

Cảnh báo: Nếu bỏ qua bước này, NLB sẽ không thể gửi traffic vào Node (Health Check Failed) vì bị tường lửa chặn.

  1. Vào AWS EC2 Console -> Instances.
  2. Chọn một Worker Node bất kỳ -> Tab Security.
  3. Bấm vào Security Group (thường có tên eks-cluster-sg-...).
  4. Chọn Edit inbound rules -> Add rule:
    • Type: Custom TCP
    • Port range: 30000-32767
    • Source: 0.0.0.0/0
  5. Bấm Save rules.

7.4. Cấu hình DNS Cuối Cùng

Lấy địa chỉ NLB của Nginx để trỏ tên miền ứng dụng về.

:: Lấy địa chỉ External IP của Nginx Service
kubectl get svc -n ingress-nginx

Copy địa chỉ EXTERNAL-IP (đuôi ...elb.amazonaws.com) và tạo CNAME trên Cloudflare:

  • Name: nlb (để thành nlb.hoatranlab.io.vn)
  • Target: Địa chỉ NLB vừa copy.
  • Proxy: Proxied (Cam).

Quy Trình CI/CD Full Stack (Harbor + ArgoCD)

8.1. Kết nối EKS với Harbor (ImagePullSecret)

Để EKS có thể vào Harbor tải ảnh về, ta phải tạo một "Chìa khóa" (Secret). Bạn hãy chạy lệnh sau trên CMD:

:: Cấu hình thông tin Harbor của bạn
:: Thay 'robot$ci' và 'mật_khẩu_robot' bằng thông tin thật bạn lấy từ Harbor

kubectl create secret docker-registry regcred ^
  --docker-server=harbor.hoatranlab.io.vn ^
  --docker-username="robot$ci" ^
  --docker-password="mật_khẩu_robot_của_bạn" ^
  --docker-email="admin@hoatranlab.io.vn" ^
  --namespace=default

8.2. Tạo File Git Repository (Cầm tay chỉ việc)

Hướng dẫn tạo thư mục:

  1. Trên Desktop (hoặc ổ D:), tạo một thư mục tên là my-app-repo.
  2. Mở thư mục đó ra. Tạo tiếp thư mục templates và .github.
  3. Trong thư mục .github, tạo thư mục con tên là workflows.

*Mẹo: Để tạo thư mục .github trên Windows, hãy đặt tên là .github. (có dấu chấm ở cuối), Windows sẽ tự sửa thành .github.

my-app-repo/
  • .github/
    • workflows/
      • ci.yml (File GitHub Actions)
  • templates/
    • deployment.yaml
    • service.yaml
    • ingress.yaml
  • Chart.yaml
  • values.yaml (File cấu hình chính)
  • Dockerfile (Code app của bạn)

1. File: .github/workflows/ci.yml

name: CI/CD Pipeline

on:
  push:
    branches: [ main ]

jobs:
  build-and-push:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    
    # Đăng nhập vào Harbor của bạn
    - name: Login to Harbor
      uses: docker/login-action@v1
      with:
        registry: harbor.hoatranlab.io.vn
        username: ${{ secrets.HARBOR_USERNAME }}
        password: ${{ secrets.HARBOR_PASSWORD }}
        
    # Build ảnh và đẩy lên Harbor
    - name: Build and Push Docker Image
      run: |
        docker build -t harbor.hoatranlab.io.vn/library/my-app:${{ github.sha }} .
        docker push harbor.hoatranlab.io.vn/library/my-app:${{ github.sha }}
        
    # Cập nhật file values.yaml để ArgoCD nhận biết ảnh mới
    - name: Update Helm Values
      run: |
        sed -i 's/tag: .*/tag: "${{ github.sha }}"/' values.yaml
        git config --global user.email "ci@hoatranlab.io.vn"
        git config --global user.name "CI Bot"
        git commit -am "Update image tag to ${{ github.sha }}"
        git push

2. File: values.yaml

image:
  repository: harbor.hoatranlab.io.vn/library/my-app
  pullPolicy: Always
  tag: "latest" # CI sẽ tự động thay số này bằng mã commit

# Tên secret tạo ở bước 8.1
imagePullSecrets:
  - name: regcred

service:
  type: ClusterIP
  port: 80

ingress:
  enabled: true
  className: "nginx"
  hosts:
    - host: nlb.hoatranlab.io.vn
      paths:
        - path: /
          pathType: Prefix

8.3. Đẩy lên GitHub & Cấu hình Secret

Bước 1: Push code

git init
git add .
git commit -m "First commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/my-app-repo.git
git push -u origin main

Bước 2: Cấu hình Secret trên GitHub

  • Vào Repo trên GitHub -> Settings -> Secrets and variables -> Actions.
  • Bấm New repository secret và thêm 2 cái:
    • HARBOR_USERNAME: (Tên robot account, vd: robot$ci)
    • HARBOR_PASSWORD: (Mật khẩu robot)

Giám Sát & Nghiệm Thu (Bước Tiếp Theo)

Sau khi bạn đã git push ở bước 8.3, quy trình tự động sẽ bắt đầu chạy. Dưới đây là các bước bạn cần làm để theo dõi và kiểm tra kết quả.

9.1. Giám sát GitHub Actions

  1. Truy cập vào Repository của bạn trên GitHub.
  2. Chuyển sang tab Actions.
  3. Bạn sẽ thấy một workflow tên là "CI/CD Pipeline" đang chạy (màu vàng).
  4. Bấm vào nó để xem chi tiết các bước: Build and Push, Update Helm Values.
  5. Khi nó chuyển sang màu xanh lá cây ✅ -> Thành công! Image đã nằm trên Harbor và file values.yaml đã được cập nhật tag mới.

9.2. Kiểm tra trên ArgoCD

  1. Đăng nhập vào ArgoCD Web UI.
  2. Vào ứng dụng my-harbor-app (mà bạn đã tạo ở bước trước, nếu chưa tạo thì hãy tạo ngay trỏ vào git repo này).
  3. Bấm nút Refresh nếu ArgoCD chưa tự động nhận diện thay đổi.
  4. Bạn sẽ thấy ArgoCD phát hiện sự thay đổi trong file values.yaml (tag image mới) và bắt đầu quá trình Sync.
  5. Trạng thái chuyển sang Healthy và Synced -> EKS đã pull image mới về và chạy Pod thành công.

9.3. Nghiệm thu Website

Cuối cùng, hãy kiểm tra thành quả của bạn.

  • Mở trình duyệt web.
  • Truy cập: https://nlb.hoatranlab.io.vn
  • Nếu bạn thấy trang "Welcome to Nginx" (hoặc nội dung file index.html bạn tạo) hiện ra cùng biểu tượng ổ khóa (SSL) an toàn.
  • CHÚC MỪNG! Bạn đã xây dựng thành công hệ thống CI/CD chuẩn doanh nghiệp.

Tổng kết hành trình:

Code -> GitHub -> GitHub Actions (Build & Push) -> Harbor -> GitHub (Update Tag) -> ArgoCD (Sync) -> EKS (Pull & Run) -> NLB (SSL) -> User.
